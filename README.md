TonForward - это Telegram-площадка, которая связывает пользователей, желающих приобрести товары из других городов или стран, с местными и путешествующими байерами, которые могут купить и переслать эти товары строго из официальных магазинов.

Для пользователей в России TonForward работает как обычный Web2-сервис: клиенты платят в рублях через классические платёжные системы, а сервис организует покупку и доставку товара. Под капотом платформа использует смарт-контракты TON только во взаимодействии между сервисом и байером - для управления эскроу, выплатами и разрешения споров - без прямого контакта конечных пользователей с криптовалютой.

Сервис поддерживает собственный пул криптовалюты, за счёт которого обеспечивает каждое рублёвое поступление и из которого выполняются выплаты байерам по условиям смарт-контрактов.

Логика сделки в TonForward:

1. Заказчик создаёт заявку: что купить, в каком официальном магазине (ссылка/домен).

2. Сервис автоматически подбирает подходящего байера (по стране/городу, KYC, нагрузке и т.д.) - без публичных карточек и без прямого контакта между заказчиком и байером.

3. Заказчик оплачивает заказ в рублях сервису (стоимость товара + вознаграждение байеру + комиссия сервиса). Сервис конвертирует нужную сумму в TON и фондирует смарт-контракт эскроу между собой и байером.

4. Байер покупает товар за свои деньги в указанном магазине, загружает фото товара и чек. После проверки сервисом байеру открывается адрес доставки (почтовый адрес или пункт выдачи), он отправляет посылку и указывает трек-номер. Вся дополнительная коммуникация (уточнения по размеру, цвету, способу доставки) проходит только через интерфейс сервиса в структурированном виде, без обмена контактами.

5. После получения посылки заказчик подтверждает доставку - эскроу выплачивает байеру вознаграждение и компенсирует покупку. Если есть проблема, заказчик открывает спор.

6. В случае спора сервис выступает арбитром и через смарт-контракт TON принимает решение: средства возвращаются сервису (для возврата заказчику) или перечисляются байеру - в зависимости от результата разбирательства и предоставленных доказательств.

Stack (MVP):

Backend: Python, Django, Django REST Framework

DB: PostgreSQL

Async / cron: Celery

Blockchain: TON, Tact smart contract Deal (escrow between platform and buyer)

Integration: TonCenter API, tonsdk

Client: Telegram Mini App / web frontend (API-driven)
